Introduction

Note:

This notation is an attempt to write mathematics in simple ASC text. The notation purposely ommits &, >, and < so as to avoid the need to escape them within an html (xml) document. Context is used in the notation and operarators are overloaded.

Current list of operators: 

	%	division

	*	multiplication, ...

	=/	"contains", "is greater than (or equal)", "is implied by", "when", "if"

	\=	"is contained by", "is less than (or equal)", "implies"

	=	"is equal to", "equals", "is equivalent to", "means", "if and only if"


	+	"or", "union", "addition"

	:	"such that"

	.	"and", used to compound functions, "intersection"

	...	used to intuit some meaning into a definition

Some reserved tokens

	N	Natural numbers

	R	Real numbers

	I[]	Identity function

"A{a}(statement)" means "For all a in A, statement", "A{a}:(statement)" means "There exists an 'a' in 'A' such that statement", and "A{a}.(statement)" means "'a' is an element of 'A' and statement".

Persitance of properties: If it is noted that an Object has some property, that property remains known as one goes down the logical chain. For example, in Y[A{a}]=B{a#a}, one avoids writing Y[A{a}]=B{A{a}#A{a}}, although one may include the redundancy for clarity. That 'a' is an element of 'A' needs only to be stated once since 'a' remains known as belonging to 'A' even after the equal sign.

Defaultion of properties: If a function is defined to take objects of certain properties, then an object is assumed to have these propeties when in context with the function. For example, in Y[A]\B.Y[a]=a#a, one avoids writing Y[A]\B.Y[A{a}]=B{a#a}, although one may include the redundancy for clarity. That 'a' is an element of 'A' is inferred from the definition of 'Y' and that 'Y' is taking on 'a'. 

 Precedence 


	0. (), {}, []	bonds content to object on the left.
	2. !		bonds to object on the right.
	1. .		bonds objects on the left and right
	2. +		''
	3. = / \ \= =/
	5. ,		''
	6. Left to right


Operator Bonding:
Operators can only bond objects of the same type, and
take on its meaning in that context.
For example,
A{a,b}A{a#b}=A{#[(a,b)]}=#[A*A]\=A
is read as
(A{a,b}A{a#b})=A{#[(a,b)]}=(#[A*A]\=A)
Note that this is equivalent to
A{a,b}(A{a#b}=A{#[(a,b)]}=(#[A*A]\=A))
This is read as
"For all 'a' and 'b' in 'A', a#b is also in 'A' and
is equivalent to a function '#' that takes an ordered pair (a,b) into 'A', and
'#' is said to be a map from A*A to A.



Promotion/Demotion of objects:
A{a,b}(A{a#b}=A{#[(a,b)]}=(#[A*A]\=A))
can be transform to
A{a,b}( (A{a#b}=A{#[(a,b)]}=#[A*A]) \=A )
by interpreting the statement A{a#b} as 
the element represented by a#b, and then
interpeting element a#b as the singleton set {a#b}.
Thus A{a#b}=({a#b}\=A), meaning that
" 'a#b is an elment of A' is equivalent to 'the singleton {a#b} is a subset of A' " 
Nonetheless, parenthesis can be used to group for clarity.

How to do differentiation and integration?
The Ruby language gives the right idea, I believe, for my notation.
Cosider sin(u(x)):

In operator mode...
	Dx[ Sin[u[x]] ] = Cos[u[x]]*Dx[u[x]]
Then, consider the following for the integral...
	Ix[ Cos[u]*Dx[u] ] = Sin[u]

In funtional mode...
	Dx[a]{ Sin[u[x]] } = Cos[u[a]]*Dx[a]{ u[x] }
	Ix[a,b]{ Cos[u[x]]*Dx[u[x]] } = Sin[u[a]] - Sin[u[b]]

In Ruby, this is termed passing a procedure to a function.
I think it works great for my notation!
